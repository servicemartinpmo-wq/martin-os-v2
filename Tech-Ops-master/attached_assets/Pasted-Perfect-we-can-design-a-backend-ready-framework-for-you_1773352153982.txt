        ↓

Key Notes:
Automated Triage: Use NLP to parse user query → map to KB articles.
Confidence Threshold: Adjustable (e.g., 0.75 for auto-suggest, <0.5 escalate).
Feedback Loop: AI learns from success/failure → updates confidence scoring.
2. Decision-Making & Logic Inventory
Logic types the system uses:
Decision Area	Logic Type	Example / Formula
Ticket Routing	Rule-based + NLP	IF confidence < 0.5 → escalate Tier1
Symptom Matching	Cosine similarity / embedding distance	similarity = cosine(query_vector, KB_vector)
Escalation Logic	Multi-tier decision tree	IF Tier1 cannot resolve in X minutes → Tier2
Resolution Suggestion Ranking	Weighted score	score = 0.6*similarity + 0.4*historical_success_rate
Knowledge Gap Detection	Frequency analysis	missing_docs = count(unresolved_patterns) / total_tickets
User Feedback Weighting	Bayesian update	new_confidence = prior_confidence * likelihood(user_feedback)
SLA Compliance	Time-bound thresholds	IF ticket_age > SLA_limit → escalate & notify
3. Formulas for Backend Decision Making
3.1 Confidence Scoring
confidence_score = (0.6 * semantic_similarity) + (0.3 * historical_success) + (0.1 * recency_weight)
semantic_similarity → NLP embedding cosine similarity of query vs KB.
historical_success → success rate of this resolution in past tickets.
recency_weight → newer resolutions get higher weight.
3.2 Escalation Timing
escalate_if = ticket_age > SLA_limit OR confidence_score < threshold
3.3 Resolution Ranking
resolution_rank = sum(for each step: success_probability * confidence_score)
Used to order suggested steps in auto-proposed solutions.
3.4 Knowledge Gap Score
knowledge_gap = unresolved_queries / total_queries_per_domain
Triggers KB creation if above threshold (e.g., 0.2 → create new KB article).
4. UDI Taxonomy for Tech Support
UDI = Unique Decision Identifier — for each ticket, AI generates a structured decision object.
Field	Type	Context	Reasoning / Decision Logic
UDI_ID	String	Unique ID	e.g., UDI-20260312-001
Domain	Enum	Networking, OS, Cloud, DB	Determines KB subset to search
Subdomain	Enum	TCP/IP, Linux, AWS	Further narrowing of KB
Symptom	Text	Extracted from user query	Used for matching
Confidence_Score	Float	AI estimated success	Formula above
Action	Enum	Suggest, Escalate, Auto-Resolve	Decision based on confidence & SLA
Decision_Reason	Text	Logic / Rules applied	e.g., “confidence < 0.5 → escalate Tier1”
Dependencies	Array	Other systems / tickets	e.g., “VPN server logs, Active Directory status”
Timestamp	DateTime	Decision creation	For tracking & SLA
Escalation_Level	Enum	Tier1–Tier4	Determined by thresholds
Feedback_Impact	Float	Weight of user feedback	Updates confidence score
